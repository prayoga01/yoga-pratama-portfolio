Brain Tumor Detection

Yoga Pratama • July 2025

This study develops a brain tumor detection system on MRI images using a Convolutional Neural Network (CNN) as a second reader, with a focus on a head-to-head comparison between the Adam and SGDM optimizers within a single controlled architecture and pipeline.

Disclaimer:This project is a personal case study.

Overview

This study develops an automated brain–tumor detection system for MRI using a Convolutional Neural Network (CNN) designed to act as a second reader for radiologists. The work addresses two persistent issues in clinical imaging workflows: (i) manual MRI interpretation is time-consuming and variable, particularly for small or low-contrast lesions; and (ii) deep learning performance in practice depends sensitively on training optimisation choices that are often under-reported. To make the findings actionable and methodologically clean, the project holds the model architecture and dataset constant and conducts a head-to-head comparison of two widely used optimisers—Adam and Stochastic Gradient Descent with Momentum (SGDM). The dataset is constructed by merging two public MRI repositories into a unified corpus with consistent labels (glioma, meningioma, pituitary, no-tumor) and harmonised preprocessing. The intended contribution is a precise, evidence-based recommendation on optimiser selection for early and accurate CNN-based detection within an MRI-centred workflow.

Main Objectives

Compare Adam vs. SGDM under identical conditions, quantifying effects on convergence dynamics (loss/accuracy curves), final test performance (accuracy, precision, recall/sensitivity, F1; optionally ROC-AUC), and stability across runs.

Methodology

Data curation and harmonisation. Two public brain-MRI sources are integrated into a single unified dataset. Class labels are mapped to a common taxonomy (glioma, meningioma, pituitary, no-tumor). Images are de-duplicated and quality-checked, then preprocessed with consistent steps: slice selection for axial views, resizing to a fixed input resolution, intensity normalisation (z-score or min–max per slice/volume), optional bias-field correction, and channel formatting (single-channel replicated to three channels if required by the backbone). To improve generalisation and balance classes, light data augmentation is applied (small rotations, translations, zoom, and intensity jitter; flips only if anatomically appropriate).

Experimental design. A single CNN backbone is fixed for all experiments (feature extractor + task-specific classification head for four classes).
